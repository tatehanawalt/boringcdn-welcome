BoringCDN Guide

Simple, fast content delivery for static sites

üì¶ What is BoringCDN?

BoringCDN is a simple content delivery network for hosting static websites and files. Upload your HTML, CSS, JavaScript, and assets - we'll handle the rest.

Upload files using an auth token, manage content from the UI.

üöÄ Quick Start

  1. Create Content: Go to your dashboard and create a new content item
  2. Create API Token: Generate a token in your settings page for uploads
  3. Upload Files: Check out the following snippets to see examples using the auth token to upload content
  4. Go Live: Your site is instantly available at your unique URL

üîë Creating an API Token

API tokens allow you to upload files programmatically. Create one in your settings page:

  1. Go to https://boringcdn.com/settings
  2. Find the "API Tokens" section
  3. Click "Create New Token"
  4. Give your token a name (e.g., "CI/CD Deploy")
  5. Copy and save the token securely - you won't see it again!

üìö Example Workflows

Deploy This Guide (Individual Files Method)

Try deploying this exact guide by uploading files individually:

# Upload index.html
curl -X POST https://boringcdn.com/api/v1/upload \
  -H "Authorization: Bearer $BORINGCDN_TOKEN" \
  -F "file=@index.html" \
  -F "content_id=$BORINGCDN_CONTENT_ID" \
  -F "path=index.html"

# Upload style.css
curl -X POST https://boringcdn.com/api/v1/upload \
  -H "Authorization: Bearer $BORINGCDN_TOKEN" \
  -F "file=@style.css" \
  -F "content_id=$BORINGCDN_CONTENT_ID" \
  -F "path=style.css"

Deploy This Guide (Tar Archive Method)

Try deploying this exact guide using a tar.gz archive:

# Create a build directory
mkdir -p dist

# Copy the guide files
cp index.html style.css dist/

# Create tar archive
tar -czf dist.tar.gz -C ./dist .

# Upload to BoringCDN
curl -X POST https://boringcdn.com/api/v1/upload \
  -H "Authorization: Bearer $BORINGCDN_TOKEN" \
  -F "file=@dist.tar.gz" \
  -F "content_id=$BORINGCDN_CONTENT_ID"

# Clean up
rm -rf dist dist.tar.gz

‚öôÔ∏è GitHub Actions Integration

Automatically deploy on every push to your repository:

.github/workflows/deploy.yml:

name: Deploy to BoringCDN

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Build
        run: npm run build  # or your build command

      - name: Create archive
        run: tar -czf dist.tar.gz -C ./dist .

      - name: Upload to BoringCDN
        run: |
          curl -X POST https://boringcdn.com/api/v1/upload \
            -H "Authorization: Bearer ${{ secrets.BORINGCDN_TOKEN }}" \
            -F "file=@dist.tar.gz" \
            -F "content_id=your-content-id" \
            --fail-with-body

      - name: Clean up
        run: rm dist.tar.gz

Note: Add BORINGCDN_TOKEN to your repository secrets in GitHub Settings ‚Üí Secrets and variables ‚Üí Actions
